# Publishing to the Official MCP Registry

Step-by-step instructions for publishing @4da/mcp-server to the official MCP Registry using the `mcp-publisher` CLI.

---

## Prerequisites

- Node.js >= 18
- npm package already published (`@4da/mcp-server` is live on npm)
- GitHub account (used for authentication)

---

## Step 1: Add mcpName to package.json

The official registry uses `mcpName` in your `package.json` to identify the server. This should match your npm package name.

Open `mcp-4da-server/package.json` and add the `mcpName` field at the top level:

```json
{
  "name": "@4da/mcp-server",
  "version": "4.0.1",
  "mcpName": "@4da/mcp-server",
  ...
}
```

**Format notes:**
- `mcpName` should match the npm package name exactly
- Scoped packages use the full scope: `@4da/mcp-server`, not just `mcp-server`
- This field tells the registry which npm package to pull metadata from

---

## Step 2: Install mcp-publisher

Install the CLI tool globally:

```bash
npm install -g mcp-publisher
```

Or use npx (no global install needed):

```bash
npx mcp-publisher --help
```

---

## Step 3: Run mcp-publisher init

From the `mcp-4da-server/` directory:

```bash
cd mcp-4da-server
npx mcp-publisher init
```

This generates a `server.json` file in your project root. The file describes your server's metadata for the registry. It will pre-populate fields from your `package.json`.

Review the generated `server.json` and ensure these fields are correct:

```json
{
  "name": "@4da/mcp-server",
  "description": "35 MCP tools for codebase-aware developer intelligence. Gives Claude, Cursor, Copilot awareness of your tech stack, dependencies, and decisions. Scores HN, arXiv, Reddit, GitHub content against what you build. Local-only, privacy-first.",
  "repository": "https://github.com/runyourempire/4DA",
  "homepage": "https://4da.ai",
  "license": "MIT",
  "runtime": "node",
  "install": {
    "command": "npx",
    "args": ["@4da/mcp-server"]
  },
  "categories": ["developer-tools"],
  "tags": ["developer-intelligence", "content-scoring", "privacy-first", "codebase-awareness"]
}
```

**Notes:**
- The exact schema of `server.json` may evolve — follow the prompts from `mcp-publisher init`
- If `init` asks questions interactively, use the values above
- The `install` block should match how users actually install the server

---

## Step 4: Login with GitHub

Authenticate with the registry using your GitHub account:

```bash
npx mcp-publisher login
```

This opens a browser window for GitHub OAuth. Once authenticated, a token is stored locally (typically in `~/.mcp-publisher/` or similar).

**If login fails:**
- Ensure you have a GitHub account that owns or has write access to the `runyourempire` org
- Check that your browser can open from the terminal
- Try `npx mcp-publisher login --manual` if browser launch fails (provides a URL to copy)

---

## Step 5: Publish

From the `mcp-4da-server/` directory:

```bash
npx mcp-publisher publish
```

This:
1. Reads your `server.json` and `package.json`
2. Validates the metadata
3. Pushes the server listing to the official MCP Registry
4. Returns a confirmation URL

**Expected output:**
```
Publishing @4da/mcp-server v4.0.1...
Validating server.json... OK
Checking npm package... OK
Publishing to MCP Registry... OK

Published: https://registry.modelcontextprotocol.io/servers/@4da/mcp-server
```

---

## Step 6: Verify

After publishing, verify the listing:

```bash
npx mcp-publisher info @4da/mcp-server
```

Also check the registry website (if one exists) to confirm the listing appears correctly.

---

## Updating the Listing

When you publish a new version to npm:

1. Update `version` in `package.json`
2. Publish to npm: `pnpm publish --access public`
3. Re-publish to registry: `npx mcp-publisher publish`

The registry should pick up the new version from npm.

---

## Troubleshooting

| Issue | Solution |
|-------|----------|
| `mcpName not found` | Add `"mcpName": "@4da/mcp-server"` to package.json |
| `Not authenticated` | Run `npx mcp-publisher login` again |
| `Package not found on npm` | Ensure `npm view @4da/mcp-server` returns data |
| `server.json not found` | Run `npx mcp-publisher init` first |
| `Version mismatch` | Ensure package.json version matches what's on npm |

## Important Notes

- The official MCP Registry is still evolving. The `mcp-publisher` CLI and `server.json` schema may change. Check https://modelcontextprotocol.io for the latest documentation.
- The registry is separate from npm — publishing to npm does not automatically list you in the MCP Registry.
- Some MCP hosts (Claude Desktop, Claude Code) may pull from the official registry for discovery. Being listed here is the highest-priority directory submission.
