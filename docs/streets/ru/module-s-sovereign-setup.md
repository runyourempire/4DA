- Может ли твой риг работать 24/7? (Питание, охлаждение, шум)
- Есть ли у тебя ИБП на случай перебоев с электричеством?

### Математика Электричества

Люди либо игнорируют это, либо катастрофизируют. Давай посчитаем по-настоящему.

**Измерение реального энергопотребления:**

```bash
# Если у тебя есть ваттметр или умная розетка с мониторингом:
# Измерь в режиме ожидания, под нагрузкой (при инференсе) и на максимуме (GPU на полной загрузке)

# Примерные оценки, если у тебя нет измерителя:
# Десктоп (без GPU, в режиме ожидания): 60-100Вт
# Десктоп (GPU среднего класса, в режиме ожидания): 80-130Вт
# Десктоп (GPU высокого класса, в режиме ожидания): 100-180Вт
# Десктоп (GPU под нагрузкой инференса): добавь 50-80% от TDP GPU
# Ноутбук: 15-45Вт
# Mac Mini M2: 7-15Вт (серьёзно)
# Ноутбук Apple Silicon: 10-30Вт
```

**Расчёт ежемесячной стоимости:**

```
Месячная стоимость = (Ватты / 1000) x Часы x Цена за кВт·ч

Пример: Десктоп с RTX 3060, инференс 8 часов/день, ожидание 16 часов/день
- Инференс: (250Вт / 1000) x 8ч x 30 дней x $0.12/кВт·ч = $7.20/месяц
- Ожидание: (100Вт / 1000) x 16ч x 30 дней x $0.12/кВт·ч = $5.76/месяц
- Итого: ~$13/месяц

Пример: Тот же риг, инференс 24/7
- (250Вт / 1000) x 24ч x 30 дней x $0.12/кВт·ч = $21.60/месяц

Пример: Mac Mini M2, 24/7
- (12Вт / 1000) x 24ч x 30 дней x $0.12/кВт·ч = $1.04/месяц
```

## Урок 2: Локальный LLM-Стек

*"Настрой Ollama для продакшн-использования — не просто для чата."*

### Почему Локальные LLM Важны для Дохода

Каждый раз, когда ты вызываешь API OpenAI, ты платишь аренду. Каждый раз, когда запускаешь модель локально, этот инференс бесплатен после начальной настройки. Математика проста:

- GPT-4o: ~$5 за миллион входных токенов, ~$15 за миллион выходных токенов
- Claude 3.5 Sonnet: ~$3 за миллион входных токенов, ~$15 за миллион выходных токенов
- Локальный Llama 3.1 8B: $0 за миллион токенов (только электричество)

Если ты строишь сервисы, обрабатывающие тысячи запросов, разница между $0 и $5-$15 за миллион токенов — это разница между прибылью и нулём.

Но вот нюанс, который большинство людей упускает: **локальные и API-модели выполняют разные роли в стеке дохода.** Локальные модели обрабатывают объём. API-модели обрабатывают критичный по качеству вывод для клиентов. Твоему стеку нужны обе.

### Установка Ollama

- Анализ тональности
- Тегирование и категоризация контента

**Производительность (типичная):**
- RTX 3060 12GB: ~40-60 токенов/секунду

- M2 Pro 16GB: ~30-45 токенов/секунду
- Только CPU (Ryzen 7 5800X): ~8-12 токенов/секунду

**Сравнение стоимости:**
- 1 миллион токенов через GPT-4o-mini: ~$0.60
- 1 миллион токенов локально (модель 8B): ~$0.003 на электричество
- Точка безубыточности: ~5,000 токенов (ты экономишь буквально с первого запроса)

**Используй для:**

- Перевод с нюансами

- Сложный анализ и рассуждения
- Генерация длинного контента
- Задачи, где качество напрямую влияет на то, заплатят тебе или нет

> **Прямой Разговор:** Если у тебя нет 24ГБ+ VRAM, пропускай модели 70B полностью. Используй вызовы API для критичного по качеству вывода. Модель 70B, работающая при 3 токенах/секунду из системной RAM, технически возможна, но практически бесполезна для любого рабочего процесса, приносящего доход. Твоё время имеет ценность.

#### Уровень 4: API-Модели (Когда Локального Недостаточно)

Локальные модели для объёма и приватности. API-модели для потолков качества и специализированных возможностей.

**Когда использовать API-модели:**
- Клиентский вывод, где качество = доход (продающие тексты, премиум-контент)
- Сложные цепочки рассуждений, на которых маленькие модели ошибаются
- Задачи с визуальными/мультимодальными данными (анализ изображений, скриншотов, документов)
- Когда нужен структурированный JSON-вывод с высокой надёжностью
- Когда важна скорость, а твоё локальное оборудование медленное

**Гибридная стратегия:**
1. Локальная LLM 7B/13B обрабатывает 80% запросов (классификация, извлечение, суммаризация)
2. API обрабатывает 20% запросов (финальная проверка качества, сложные задачи, клиентский вывод)
3. Твоя эффективная стоимость: ~$0.50-2.00 за миллион токенов в среднем (вместо $5-15 чисто API)

Этот гибридный подход — как ты строишь сервисы со здоровой маржинальностью. Подробнее в Модуле R.

## Урок 3: Преимущество Приватности

*"Твоя приватная настройка — ЭТО конкурентное преимущество, а не просто предпочтение."*

### Приватность — Это Характеристика Продукта, Не Ограничение

Большинство разработчиков настраивают локальную инфраструктуру, потому что лично ценят приватность или любят экспериментировать. Это нормально. Но ты упускаешь деньги, если не осознаёшь, что **приватность — одна из самых востребованных характеристик в технологиях прямо сейчас.**

Вот почему: каждый раз, когда компания отправляет данные в API OpenAI, эти данные проходят через третью сторону. Для многих бизнесов — особенно в здравоохранении, финансах, юриспруденции, госструктурах и компаниях из ЕС — это реальная проблема. Не теоретическая. Проблема уровня "мы не можем использовать этот инструмент, потому что комплаенс запретил".

Ты, запуская модели локально на своей машине, этой проблемы не имеешь.

### Нормативный Попутный Ветер

Нормативная среда движется в твоём направлении. Быстро.

**EU AI Act (применяется с 2024-2026):**
- Системы ИИ высокого риска требуют документированных пайплайнов обработки данных
- Компании должны демонстрировать, куда текут данные и кто их обрабатывает
- Локальная обработка радикально упрощает соответствие
- Компании ЕС активно ищут провайдеров ИИ-сервисов, которые могут гарантировать резидентность данных в ЕС

**GDPR (уже применяется):**
- "Обработка данных" включает отправку текста в API LLM
- Компании нуждаются в Соглашениях об Обработке Данных с каждой третьей стороной
- Локальная обработка полностью устраняет третью сторону
- Это реальное преимущество: "Ваши данные никогда не покидают вашу инфраструктуру. Нет никакого DPA с третьей стороной для согласования."

### Как Позиционировать Приватность в Переговорах о Продаже

Тебе не нужно быть экспертом по комплаенсу. Тебе нужно понимать три фразы и знать, когда их использовать:

**Фраза 1: "Ваши данные никогда не покидают вашу инфраструктуру."**
Используй когда: Говоришь с любым потенциальным клиентом, заботящимся о приватности. Это универсальный крючок.

**Фраза 2: "Не требуется соглашение об обработке данных с третьей стороной."**
Используй когда: Говоришь с европейскими компаниями или любой компанией с юридическим/комплаенс-отделом. Это экономит им недели юридической проверки.

**Фраза 3: "Полный аудиторский след, обработка на выделенных ресурсах."**
Используй когда: Говоришь с корпоративными клиентами или регулируемыми отраслями. Им нужно доказать свой ИИ-пайплайн аудиторам.

**Обоснование Премиальной Цены:**

**Стандартный сервис ИИ-обработки (через облачные API):**
- Данные клиента идут в OpenAI/Anthropic/Google
- Ты конкурируешь с каждым разработчиком, способным вызвать API
- Рыночная ставка: $0.01-0.05 за обработанный документ

**Сервис ИИ-обработки с приоритетом приватности (твой локальный стек):**
- Данные клиента остаются на твоей машине
- Ты конкурируешь с гораздо меньшим пулом провайдеров
- Рыночная ставка: $0.10-0.50 за обработанный документ (премия 5-10x)
- Ты продаёшь инфраструктуру + экспертизу + соответствие

Премия за приватность реальна: **от 5x до 10x** по сравнению с товарными облачными сервисами за ту же задачу.

*"У твоего бизнеса есть burn rate. Знай его. Контролируй его. Заставь работать."*

### Почему {= regional.currency_symbol | fallback("$") =}200/месяц

Двести {= regional.currency | fallback("долларов") =} в месяц — это минимально жизнеспособный бюджет для операции разработчика по генерации дохода. Этого достаточно для реальных сервисов, реальных клиентов и реального дохода. И достаточно мало, чтобы если ничего не работает, ты не потерял всё.

Цель проста: **превратить {= regional.currency_symbol | fallback("$") =}200/месяц в {= regional.currency_symbol | fallback("$") =}600+/месяц в течение 90 дней.** Если получится — у тебя бизнес. Если нет — меняй стратегию, не увеличивай бюджет.

### Расчёт ROI

Это единственное число, которое имеет значение:

```
Месячный Доход - Месячные Расходы = Чистая Прибыль
Чистая Прибыль / Месячные Расходы = Мультипликатор ROI

Пример:
$600 дохода - $200 расходов = $400 прибыли
$400 / $200 = 2x ROI

Цель: 3x ROI ($600+ дохода на $200 расходов)
Минимум: 1x ROI ($200 дохода = безубыточность)
Ниже 1x: Меняй стратегию или сокращай расходы
```

{@ insight cost_projection @}

> **Типичная Ошибка:** Относиться к {= regional.currency_symbol | fallback("$") =}200 как к "инвестиции", которая не должна возвращать деньги немедленно. Нет. Это эксперимент со сроком 90 дней. Если {= regional.currency_symbol | fallback("$") =}200/месяц не генерируют {= regional.currency_symbol | fallback("$") =}200/месяц дохода в течение 90 дней, что-то в стратегии нужно менять. Будь честен с собой.

### Чекпоинт Урока 5

Ты должен теперь иметь:
- [ ] Месячный бюджет ~$200, распределённый по четырём уровням
- [ ] API-аккаунты с настроенными лимитами расходов
- [ ] Принятые решения по инфраструктуре (только локально vs. локально + VPS)
- [ ] Выбранный стек инструментов (в основном бесплатные планы для старта)
- [ ] Цели ROI: 3x в течение 90 дней
- [ ] Чёткое правило: увеличивай бюджет только после доказательства ROI

### Как Использовать Этот Документ

1. **Перед началом любого нового проекта:** Проверь свой Суверенный Стек. Достаточно ли оборудования, времени, навыков и бюджета для реализации?
2. **Перед покупкой чего-либо:** Проверь распределение бюджета. Эта покупка в плане?

4. **Когда кто-то спрашивает, чем ты занимаешься:** Раздел "Что Я Могу Предложить Сегодня" — твой мгновенный питч.
5. **Когда соблазняешься погнаться за новой блестящей идеей:** Проверь свои ограничения.

> **Типичная Ошибка:** Потратить 3 часа на "идеальный" документ вместо 1 часа на "готовый." Документ Суверенного Стека — рабочий справочник, не бизнес-план для инвесторов. Никто его не увидит кроме тебя. Точность важна. Форматирование — нет.
